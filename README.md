# Palworld 中文可视化配置工具

一款面向 Palworld 私人服务器玩家的本地可视化配置工具。

它可以将难以阅读和修改的 `PalWorldSettings.ini` 转换为中文表单，让用户通过开关、输入框、滑块和选项直接编辑服务器参数，并重新生成完整配置文件。

当前版本：

```text
v0.1.0 Beta
```

---

## 项目简介

Palworld 私人服务器的配置通常集中在一段很长的 `OptionSettings=(...)` 中。

例如：

```ini
[/Script/Pal.PalGameWorldSettings]
OptionSettings=(ExpRate=1.000000,PalCaptureRate=1.000000,...)
```

直接编辑这类配置时，用户通常需要：

* 查找英文参数含义
* 判断布尔值、倍率和枚举值
* 手动处理引号、逗号和括号
* 避免破坏原有未知参数
* 判断不同配置之间是否冲突
* 评估高负载配置对服务器性能的影响

本工具希望将这个过程简化为：

```text
导入现有配置
→ 自动解析
→ 中文可视化编辑
→ 实时校验
→ 查看修改差异
→ 应用体验预设
→ 生成完整配置
→ 复制或下载
```

## 当前功能

### 配置导入

支持以下导入方式：

* 粘贴完整 `PalWorldSettings.ini`
* 粘贴单独的 `OptionSettings=(...)`
* 导入 `.ini` 文件
* 自动提取并解析 `OptionSettings`
* 自动回填中文可视化表单

解析器支持：

* 字符串
* 数字
* 布尔值
* 枚举
* 数组
* 带逗号的引号内容
* 括号嵌套内容

---

### 中文可视化编辑

参数由统一 Schema 管理，并根据参数类型生成对应控件。

当前控件包括：

* 开关
* 数字输入框
* 滑块
* 无限值数字输入
* 分段选择
* 文本输入
* 密码输入
* 多选输入

每个参数可以包含：

* 中文名称
* 英文参数 Key
* 参数说明
* 示例
* 默认值
* 最小值和最大值
* 推荐范围
* 单位
* 精度
* 分类
* 依赖关系

---

### 简单模式与专业模式

工具支持两种编辑模式：

```text
简单模式
专业模式
```

简单模式展示常用配置。

专业模式展示更多服务器、网络、权限、性能、公会和实验性参数。

---

### 搜索与筛选

支持：

* 搜索中文名称
* 搜索英文参数 Key
* 搜索参数说明
* 按分类筛选
* 只查看已修改参数

---

### 配置校验

工具会实时检查参数值和参数之间的关系。

#### 参数值校验

包括：

* 类型是否合法
* 数字是否有效
* 是否超过硬限制
* 是否超出推荐范围
* 必填内容是否为空
* 枚举值是否合法
* 数组内容是否合法

校验结果分为：

```text
error
```

错误会阻止配置导出。

```text
warning
```

提醒不会阻止配置导出。

#### 参数关系校验

当前包括：

* 合作人数不能超过服务器人数
* 据点数量关系检查
* PvP 相关配置联动
* 语音聊天相关配置联动
* RCON 与 REST API 配置联动
* 公会自动清理风险检查
* 随机模式相关配置检查
* 据点、同步距离和帕鲁数量的性能风险检查
* 掉落物数量与服务器负载检查

---

### 未知参数保留

工具不会删除无法识别的参数。

未知参数会以原始 Key 和原始值保存，并在重新生成配置时继续写回。

这是项目中的重要原则：

> 工具不认识某个参数，不代表这个参数无效。

---

### 体验预设

当前提供四套服务器体验预设：

#### 休闲建造

适合偏建造和轻度生存的服务器。

主要调整：

* 降低生存消耗
* 降低建筑劣化
* 提高采集效率
* 提高工作效率
* 降低袭击压力
* 减少死亡惩罚

#### 好友合作

适合小型好友服务器长期游玩。

主要调整：

* 保留适度挑战
* 加快经验成长
* 提高捕获效率
* 提高采集效率
* 缩短孵化等待时间

#### 快速成长

适合希望快速进入中后期内容的服务器。

主要调整：

* 提高经验倍率
* 提高工作速度
* 提高采集与掉落
* 缩短孵化时间
* 缩短部分事件等待时间

#### 性能优先

适合配置较低或人数较多的服务器。

主要调整：

* 降低帕鲁生成密度
* 降低掉落物数量
* 降低据点规模
* 降低同步距离
* 减少服务器持续负载

预设只会修改与体验目标相关的参数。

以下内容不会被预设覆盖：

* 服务器名称
* 服务器描述
* 服务器密码
* 管理员密码
* 最大玩家人数
* 公网 IP 和端口
* RCON 设置
* REST API 设置

---

### 预设应用前校验

应用预设前，工具会比较：

* 当前已有问题
* 应用预设后的问题
* 预设新增的问题
* 预设解决的问题
* 应用后仍存在的问题

规则如下：

```text
预设新增 error
→ 禁止应用
```

```text
已有 error，但预设没有新增 error
→ 允许应用并提示
```

```text
预设解决已有问题
→ 显示已解决的问题
```

---

### 修改差异

工具支持查看：

* 当前值
* 最近导入值
* 官方默认值
* 修改前完整 INI
* 修改后完整 INI
* 已修改参数列表

密码内容会在差异视图中脱敏。

需要注意：

```text
最近导入值
≠
官方默认值
```

用户导入配置后，单项撤销和全部撤销默认回到最近导入状态，而不是强制恢复官方默认。

---

### 本地草稿

工具会在用户修改配置后自动保存本地草稿。

流程：

```text
修改配置
→ 自动保存到 localStorage
→ 刷新或重新打开页面
→ 提示恢复草稿
```

草稿会保存：

* 当前参数值
* 最近导入基准
* 未知参数
* 导入来源
* 保存时间
* 草稿版本

出于安全考虑，以下内容不会进入本地草稿：

```text
ServerPassword
AdminPassword
```

密码仍然可以正常写入最终导出的 INI 文件。

所有草稿仅保存在用户本地浏览器中，不会上传到服务器。

---

### 配置生成与导出

支持：

* 生成完整 `PalWorldSettings.ini`
* 正确处理字符串引号
* 正确处理布尔值
* 正确处理数字
* 正确处理数组
* 保留未知参数
* 复制生成结果
* 下载 `.ini` 文件

当存在 `error` 时，下载和导出会被阻止。

当只有 `warning` 时，用户仍然可以导出配置。

## 设计原则

本项目开发过程中遵循以下原则：

1. 最近导入值和官方默认值必须分开。
2. 未知参数不能被自动删除。
3. 密码不能进入本地自动草稿。
4. 密码必须能够正常进入最终导出配置。
5. 体验预设只修改相关参数。
6. 预设不能覆盖整份用户配置。
7. 错误阻止导出，提醒不阻止导出。
8. 所有配置在本地处理。
9. 功能完成后优先测试和交付。
10. 桌面端和 Windows EXE 是最终交付方向。

## 注意事项

在修改真实服务器配置前，建议先备份原始文件。

常见配置文件位置会因为服务器安装方式而有所不同，请根据自己的服务器目录确认实际路径。

工具会尽量保留未知参数，但仍建议在每次修改前保留原始备份。

本项目仍处于 Beta 阶段，请在正式服务器环境中谨慎使用生成的配置。
